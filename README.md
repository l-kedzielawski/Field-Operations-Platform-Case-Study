# Northfield Ops Platform

This git is the portfolio version of the project documentation for a custom internal platform built for a field operations business.

The original company and product name are intentionally hidden. In this case study, the platform is presented under the working name `Northfield Ops`.

The goal of this docs set is simple: explain the project in a way that feels credible, readable, and worth spending time on.


## What this project is

`Northfield Ops` is a custom operations platform for a 200+ person field workforce working across Poland and Germany.

Before the platform, the business relied on chat messages, spreadsheets, calls, and manual follow-up. That made approvals slow, compliance easy to miss, and attendance records difficult to trust.

The system brought those moving parts into one place:

- field reporting and approvals,
- document compliance and expiry tracking,
- attendance and time records,
- and map-based operational visibility.

## Why it is a strong portfolio piece

This is not interesting because it has a lot of screens.

It is interesting because it shows the ability to:

- understand a messy real-world workflow,
- design rules that the product actually enforces,
- work pragmatically inside an existing client stack,
- and ship something people can use every day.

## Outcome snapshot

| Metric | Before | After |
|--------|--------|-------|
| Report approval cycle | 48 hours | 12 hours |
| Expired document incidents | 6 per month | 0 per month |
| Payroll correction rate | 18% | under 5% |
| Admin coordination overhead | high manual effort | lower by about 20 hours per week |
